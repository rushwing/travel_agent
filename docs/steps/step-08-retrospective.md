# Step 8: Retrospective

## Goal

Close the tutorial by writing down what worked, what failed, and what should change next.

## Basic knowledge

- A tutorial project is more valuable when it leaves behind design reasoning, not only code.
- The final README should explain the current architecture and the main tradeoffs.
- A short retrospective is enough if it is concrete.

## What to update

- `README.md`: add a summary of the final architecture, strongest lessons, and next steps.
- `docs/lessons_learned.md`: append the hardest bug you fixed.
- `docs/future_todo.md`: promote your best stretch ideas.

## Guiding question

If you rebuilt this project in one weekend, what would you keep, what would you simplify, and what would you postpone?
